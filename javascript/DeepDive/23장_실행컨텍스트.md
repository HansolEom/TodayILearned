# 23장. 실행 컨텍스트
## 23.1 소스코드의 타입
ECMAScript 사양은 소스코드를 4가지 타입으로 구분한다
|소스코드의 타입|설명|
|---|---|
|전역 코드|전역에 존재하는 소스코드를 말한다. 클래스 등의 내부 코드는 포함되지 않는다.|
|함수 코드|함수 내부에 존재하는 코드로 중첩함수, 클래스 등의 내부 코드는 포함되지 않는다.|
|eval 코드|빌트인 전역 함수인 eval 함수에 인수로 전달되어 실행되는 소스코드를 말한다.|
|모듈 코드|모듈 내부에 존재하는 소스코드로 모듈 내부의 함수, 클래스 등의 내부 코드는 포함되지 않는다.|  

소스코드를 4가지 타입으로 구분하는 이유는 타입에 따라 실행 컨텍스트를 생성하는 과정과 관리 내용이 다르기 떄문이다.
1. 전역코드:   
  전역 변수를 관리하기 위해서는 최상위 스코프인 전역 스코프를 생성해야 하고 var 키워드로 선언된 전역 변수와 함수 선언문으로 정의된 전역 함수를 전역 객체의 프로퍼티와 메서드로 바인딩하고 참조하기 위해 전역 객체와 연결되야 한다. 이를 위해 전역 코드가 평가되면 전역 실행 컨텍스트가 생성된다.
2. 함수 코드:   
  함수 코드는 지역 스코프를 생성하고 지역변수, 매개변수, arguments 객체를 관리해야한다. 생성한 지역 스코프를 전역 스코프에서 시작하는 스코프 체인의 일원으로 연결해야 하고 이를 위해 함수 코드가 평가되면 함수 실행 컨텍스트가 생성된다.
3. eval 코드:   
  eval 코드는 strict mode에서 자신만의 독자적 스코프를 생성한다. 이를 위해 eval 코드가 평가되면 eval 실행 컨텍스트가 생성된다.
4. 모듈 코드:   
   모듈 코드는 모듈별로 독립적인 모듈 스코프를 생성한다. 모듈 코드가 평가되면 모듈 실행 컨텍스트가 생성된다.
## 23.2 소스코드의 평가와 실행
자바스크립트 엔진은 소스코드를 평가와 실행 2가지 과정으로 나누어 처리한다. 평가 과정에서는 실행 컨텍스트를 생성하고 변수, 함수등의 선언문만 먼지 생성하여 생성된 변수나 함수 식별자를 키로 실행 컨텍스트가 관리하는 스코프에 등록한다.   
평가 과정이 끝나면 선운문을 제외한 소스코드가 순차적으로 실행된다. 소스코드 실행에 필요한 정보(변수, 함수의 참조)를 실행 컨텍스트가 관리하는 스코프에서 검색해 가져온다. 변수 값의 변경등 소스코드의 실행 결과는 다시 실행 컨텍스트가 관리하는 스코프에 등록된다.
```js
var x
x = 1
```
자바스크립트 엔진은 먼저 평가 과정을 통해 변수 선언문 var x를 먼저 실행한다. 생성된 변수 x는 실행 컨텍스트가 관리하는 스코프에 등록되고 undefined로 초기화 된다.   
평가 과정이 끝나면 소스코드 실행 과정이 시작된다. var x 선언문은 소스코드 평가 과정에서 이미 실행 완료 되었다. 소스코드 실행 과정에서는 변수 할당문 x = 1만 실행되고 이때 x 변수에 값을 할당하려면 x가 선언된 변수인지 먼저 확인해야한다.   
실행 컨텍스트가 관리하는 스코프에 x 변수가 등록 되어있는지 확인하고 등록 되어 있다면 선언된 변수에 값을 할당하고 할당 결과를 실행 컨텍스트에 등록한다.
## 23.3 실행 컨텍스트의 역할
```js
// 전역 변수 선언
const x = 1
const y = 2

// 함수 정의
function foo(a) {
  // 지역 변수 선언
  const x = 10
  const y = 20

  // 메서드 호출
  console.log(a + x + y) // 130
}

// 함수 호출
foo(100)

// 메서드 호출
console.log(x + y) // 3
```
위 소스코드의 실행 과정을 살펴보자
1. 전역 코드 평가:   
  평가 과정을 통해 코드를 실행하기 위한 준비를 한다. 소스코드 평가 과정에서 선언문이 먼저 실행되기 때문에 변수 선언문과 함수 선언문이 먼저 실행되고, 그결과 생성된 전역 변수와 전역 함수를 실행 컨텍스트가 관리하는 전역 스코프에 등록한다. 이떄 var로 선언한 전역 변수와 함수 선언문으로 정의한 전역 함수는 전역 객체의 프로퍼티와 메서드가 된다.
2. 전역 코드 실행:   
  런타임이 시작되며 전역 코드가 순차적으로 실행된다. 전역 변수에 값이 할당되고 함수가 호출된다. 함수가 호출되면 전역 코드의 실행을 일시 중단하고 코드 실행 순서를 변경하여 함수 내부로 진입한다.
3. 함수 코드 평가:   
  함수 호출에 의해 실행 순서가 변경되어 함수 내부로 진입하면 함수 내부의 문들을 실행하기 앞서 함수 코드 평가 과정을 거친다. 이떄 매개 변수와 지역 변수 선언문이 먼저 실행되고 그 결과를 매개변수와 지역 변수가 실행 컨텍스트가 관리하는 지역 스코프에 등록한다. 함수 내부에서 사용 할 수 있는 arguments 객체가 생성되어 지역 스코프에 등록되고 this 바인딩도 결정된다.
4. 함수 코드 실행:   
  평가가 끝나면 런타임 되어 함수 코드가 순차적으로 실행된다. 매개변수와 지역 변수에 값이 할당되고 console.log 메서드가 호출된다.   
  console.log 메서드를 호출하기 위해 console을 스코프 체인을 통해 검색한다. 함수 코드의 지역 스코프는 상위 스코프인 전역 스코프와 연결되야 하지만 console 식별자는 스코프 체인에 등록되어 있지 않고 전역 객체에 프로퍼티로 존재한다. 이는 전역 객체의 프로퍼티가 전역 변수처럼 전역 스코프를 통해 검색 가능하다는 것을 의미한다.   
  log 프로피트를 console 객체의 프로토타입 체인을 통해 검색하고 console.log 메서드에 인수로 전달된 표현식 a + x + y가 평가된다. 스코프 체인을 통해 식별자를 검색하고 console.log 메서드가 종료되면 함수 실행이 종료되고 함수 호출 이전으로 돌아가 전역 코드 실행을 이어간다. 
**이처럼 코드가 실행되려면 다음과 같이 스코프, 식별자, 코드 실행 순서등의 관리가 필요하다.**
1. 선언에 의해 생성된 모든 식별자를 스코프를 구분하여 등록하고 상태 변화를 지속적으로 관리할 수 있어야 한다.
2. 스코프는 중첩 관계에 의해 상위 스코프로 이동하여 색별자를 검색할 수 있는 스코프 체인을 형성해야 한다. 
3. 현재 실행중인 코드의 실행 순서를 변경할 수 있어야 하며 되돌아 갈수도 있어야 한다.   

**실행 컨텍스트는 소스코드를 실행하는데 필요한 환경을 제공하고 코드의 실행 결과를 실제로 관리하는 영역이다. 식별자를 등록하고 관리하는 스코프와 코드 실행 순서 관리를 구현한 내부 메커니즘으로 모든 코드는 실행 컨텍스트를 통해 실행되고 관리된다.** 식별자와 스코프는 실행 컨텍스트의 렉시컬 환경으로 관리하고 코드 실행 순서는 실행 컨텍스트 스택으로 관리한다.
## 23.4 실행 컨텍스트 스택
```js
const x = 1

function foo () {
  const y = 2

  function bar () {
    const z = 3
    console.log(x + y + z)
  }
  bar()
}

foo() // 6
```
위 코드는 소스코드 타입으로 전역 코드와 함수 코드를 가지고 있다. 자바스크립트 엔진은 전역 코드를 평가해 전역 실행 컨텍스트를 생성하고 함수가 호출되며 함수 코드를 평가해 함수 실행 컨텍스트를 생성한다.   
이때 실행컨텍스트는 스택 자료구조로 실행 컨텍스트 스택이라 부른다.
1. 전역 코드의 평가와 실행:   
  자바스크립트 엔진이 전역 코드를 먼저 평가해 전역 실행 컨텍스트를 생성하고 실행 컨텍스트 스택에 푸쉬한다. x와 foo는 전역 실행 컨텍스트에 등록 후 전역 코드가 실행되기 시작하여 전역 변수 x 값이 할당되고 foo가 호출된다.
2. foo 함수 코드의 평가와 실행:   
  foo가 호출되면 전역 코드의 실행은 중단되고 코드의 제어권이 foo 함수 내부로 이동한다. 자바스크립트 엔진은 goo 함수 내부의 함수 코드를 평가해 함수 실행 컨텍스트를 생성하고 실행 컨텍스트 스택에 푸쉬한다. 이때 지역 변수 y와 중첩함수 bar가 함수 실행 컨텍스트에 등록되고 foo 함수 코드가 실행 될 때 변수 y값이 할당되고 중첩합수 bar가 호출된다.
3. bar 함수 코드의 평가와 실행:   
  bar가 호출되면 foo 함수 코드의 실행은 일시 중단되고 제어권이 bar 함수 내부로 간다. 자바스크립트 엔진은 bar 함수 코드를 평가하고 함수 실행 컨텍스트를 생성하고 실행 컨텍스트 스택에 푸쉬한다. 지역변수 z가 함수 실행컨텍스트에 등록되고 함수가 실행되면 변수 z에 값이 할당되고 console.log를 호출한 후 bar함수는 종료된다.
4. foo 함수 코드로 복귀:
  bar 함수가 종료되면 제어권이 foo로 다시 이동하고 실행컨텍스트 스택에서 팝하여 bar함수 실행 컨텍스트를 제거한다. 그리고 foo함수가 종료된다.
5. 전역 코드로 복귀:
  foo가 종료되면 제어권이 다시 전역 코드로 이동하고 자바스크립트 엔진은 실행 컨텍스트 스택에서 팝하여 foo 실행 컨텍스트를 제거한다. 더이상 전역 코드가 남아 있지 않으면 실행 컨텍스트 스택에서 전역 실행 컨텍스트를 팝하고 스택에 아무것도 남아 있지 않게 된다.   

**이처럼 실행 컨텍스트 스택은 코드의 실행 순서를 관리한다. 실행 컨텍스트 스택 최상위에 존재하는 실행 컨텍스트는 언제나 실행중인 실행 컨텍스트라 부른다.**
## 23.5 렉시컬 환경
렉시컬 환경은 식별자와 식별자에 바인딩 값, 상위 스코프에 대한 참조를 기록하는 자료구조로 실행 컨텍스트를 구성하는 컴포넌트다.   
렉시컬 환경은 키와 값을 갖는 객체 형태의 스코프를 생성하여 식별자를 키로 식별자에 바인딩 된 값을 관리한다. 이떄 식별자를 등록하고 관리하는 저장소 역할을 하는 렉시컬 스코프의 실체다.   
실행 컨텍스트는 LexicalEnvironment 컴포넌트와 VariableEnvironment 컴포넌트로 구성된다.   
생성 초기에는 두개의 컴포넌트는 하나의 동일한 렉시컬 환경을 참조하다 특정 상황을 만나면 VariableEnvironment 컴포넌트를 위한 새로운 렉시컬 환경을 만들고 두개의 컴포넌트 내용이 달라지는 경우도 있다.   
렉시컬 환경은 다음과 같이 두개의 컴포넌트로 구성된다.
1. 환경 레코드:   
  스코프에 포함된 식별자를 등록하고 등록된 식별자에 바인딩된 값을 관리한느 저장소로 소스코드의 타입에 따라 관리하는 내용에 차이가 생긴다.
2. 외부 렉시컬 환경에 대한 참조:
  외부 렉시컬 환경에 대한 참조는 외부 렉시컬 환경인 상위 스코프를 가지고 이는 실행 컨텍스트를 생성한 소스코드를 포함하는 상위 코드의 렉시컬 환경을 말한다. 외부 렉시컬 환경에 대한 참조를 통해 단방향 링크드 리스크인 스코프 체인을 구현한다.
## 23.6 실행 컨텍스트의 생성과 식별자 검색 과정
```js
var x = 1
const y = 2

function foo (a) {
  var x = 3
  const y = 4

  function bar (b) {
    const z = 5
    console.log(a + b + x + y + z)
}
  bar(10)
}

foo(20) // 42
```
### 23.6.1 전역 객체 생성
전역 객체는 전역 코드가 평가되기 이전에 생기는데 전역 객체에는 빌트인 전역 프로퍼티와 빌트인 전역함수, 표준 빌트인 객체가 추가되며 동작환경에 따라 클리이언트 사이드 Web API 또는 특정 환경을 위한 호스트 객체를 포함한다.
```js
// Object.prototype.toString
window.toString() // -> "[object Window]"

window.__proto__.__proto__.__proto__.__proto__ === Object.prototype // -> true
```
전역 객체도 Object.prototype을 상속 받는다. 전역 객체도 프로토타입 체인의 일원이다.
### 23.6.2 전역 코드 평가
소스코드가 로드되면 자바스크립트 엔진은 전역 코드를 평가한다.

2. 전역 렉시컬 환경 생성:

### 23.6.3 전역 코드 실행
전역 코드가 순차적으로 실행되며 변수 할당문이 실행되 변수에 값이 할당되고 함수가 호출된다.   
변수 할당 또는 함수를 호출하기 위헤서는 선언된 식별자인지 확인하고 선언되지 않았을 경우 참조할 수 없으므로 할당이나 호출도 할수 없다. 스코프를에 따라 같은 식별자가 존재할 수 있어 어느 스코프를 참조해야하는지 결정할 필요가 있는데 이를 **식별자 결장**이라 한다. **식별자 결정을 위해 식별자를 검색할 떄는 실행중인 실행 컨텍스트에서 식별자를 검색하기 시작한다.**   
전역 레시컬 환경에서 변수와 함수를 검색하기 시작한다. 실행 컨텍스트의 렉시컬 환경에서 식별자를 검색할 수 없으면 외부 렉시컬 환경에 대한 참조가 가리키는 렉시컬 환경, 즉 상위 스코프로 이동하여 식별자를 검색한다.   
렉시컬 환경은 스코프 체인의 종점으로 전역 레시컬 환경에서 검색할수 없는 식별자는 참조 에러를 발생한다.

### 23.6.5 foo 함수 코드 실행
러타임이 시작되어 foo함수가 순차적으로 실행되고 매개 변수에 인수가 할당되고 변수 할당문이 시행되 변수가 할당된다. 그리고 bar함수가 호출 된다.   
**식별자 결정을 위해 실행중인 실행 컨텍스트의 렉시컬 환경에서 식별자를 검색하기 시작한다.**  foo 함수 렉시컬 환경에서 식별자를 검색하기 시작한다 만약 검색할 수 없으면 외부 렉시컬 환경에 대한 참조가 가리키는 렉시컬 환경으로 이동하여 식별자 검색을 한다. 검색된 식별자에 값을 바인딩 한다.
### 23.6.6 bar 함수 코드 평가
bar함수를 호출하면 bar 함수 내부로 제어권이 이동하고 함수를 평가하기 시작한다. 실행컨텍스트와 렉시컬 환경 생성과정은 foo와 동일하다.
### 23.6.7 bar 함수 코드 실행
런타임에 bar의 소스 코드가 순차적으로 실행되며 매개변수에 인수가 할당되고 변수 할당문이 실행되어 z에 값이 할당된다.
1. console.log 식별자 검색:   
  스코프 체인에서 console 식별자를 검색한다. bar -> foo -> 전역 렉시컬 환경으로 이동해 console 식별자를 검색한다. console 식별자는 객체 환경 레코드의 BindingObject를 통해 전역 객체에서 찾을 수 있다.
2. log 메서드 검색:   
  바인딩된 console에서 log를 검색한다. 이때 console 객체의 프로토 타입 체인을 통해 메서드를 검색하고 log 메서드는 상속된 프로퍼티가 아닌 console 객체가 직접 소유한 프로퍼티다.
3. 표현식 a + b + x + y + z 평가:
  전달된 a + b + x + y + z를 평가하기 위해 a,b,x,y,z식별자를 검색한다. a 식별자는 foo, b 식별자는 bar, x와 y 식별자는 foo, z 식별자는 bar 함수 렉시컬 환경에서 검색된다.
4. console.log 메서드 호출:   
  생성된 값을 console.log 메서드에 전달하여 호출한다.
### 23.6.8 bar 함수 코드 실행 종료
consol.log 메서드가 호출되고 종료하면 실행할 코드가 없으므로 bar 함수는 실행이 종료되고 실행 컨텍스트 스택에서 bar 실행 컨텍스트가 팝되어 제거되고 foo 실행 컨텍스트가 실행중인 실행 컨텍스트가 된다.
### 23.6.9 foo 함수 코드 실행 종료
bar 함수가 종료되면 foo 함수 코드도 더이상 실행될 코드가 없으므로 실행이 종료된다. 실행 컨텍스트 스택에서 팝되고 전역 실행 컨텍스트가 실행중인 실행 컨텍스트가 된다. 
### 23.6.10 전역 코드 실행 종료
전역 코드가 종료되고 전역 실행 컨텍스트도 실행 컨텍스트 스택에서 팝되 아무것도 남지 않게 된다.
## 23.7 실행 컨텍스트와 블록 레벨 스코프
let, const키워드로 선언한 변수는 코드 블록을 지역 스코프로 인정하는 블록 레벨 스크프를 따른다. 
```js
let x = 1

if (true) {
  let x = 10
  console.log(x) // 10
}

console.log(x) // 1
```
if문의 코드 불록 내에서 let으로 선언된 변수는 블록레벨 스코프를 생성해야 한다. 이를 위해 선언적 환경 레코드를 갖는 렉시컬 환경르 새롭게 생성하여 기존의 전역 렉시컬 환경을 교체한다. 새롭게 생성된 렉시컬 환경의 외부 렉시컬 환경에 대한 참조는 if문이 실행되기 이전의 전역 렉시컬 환경을 가리킨다.   
모든 블록 레벨 스코프를 생성하는 블록문에 적용된다. for문의 변수 선언문을 let으로 할 경우 반복실행 마다 새로운 렉시컬 환경을 만든다. for문 안에서 만든 함수가 있다면 함수의 상위 스코프는 for문의 코드 블록이 생성한 렉시컬 환경이다.   
이때 유지해야 하는 ㄱ밧은 독립적 레시컬 환경을 생성해 식별자 값을 유지한다. 이는 클로저에서 살펴본다.
